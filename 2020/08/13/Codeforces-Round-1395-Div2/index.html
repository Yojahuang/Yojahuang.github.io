Codeforces Round 1395(Div2)

Author: Yojahuang Date: 八月 13, 2020  16:22:31 Category: 競程

前言

在打這場時，靠著假解拿到蠻前面的名次
不過system test後就掉下去了
依我的狀況來說 能一次打五題其實很難得 所以就記錄一下吧

A

給你 a 個 ‘a’ 字元 b 個 ‘b’ 字元 c 個 ‘c’ 字元 d 個 ‘d’ 字元

你可以進行以下操作無限次

將 a b c 各 -1 然後將 d 替換成 d + 3

問經過任意次操作後 是否能將字元排成回文

$(0 \leq a, b, c, d \leq 10^9)$

如果要排出回文的話， abcd最多只能有一個是奇數
所以我們可以分別檢查 $(a, b, c, d)$ 以及 $(a - 1, b - 1, c - 1, d + 3)$ 的奇偶性
這樣這題就做完了

B

有一個 $n \times m$ 的二維Grid，沒有任何障礙物

你有一個棋子在 $(x, y)$ 上，而棋子的移動規則跟西洋棋的城堡一樣

請你構造出一種走法 使得你可以經過每個點剛好一次且 $(x, y)$ 是起點

$(3 \leq n,m \leq 100, 2 \leq x \leq n - 1), (2 \leq y \leq m - 1)$

一開始沒有看到要令 $(x, y)$ 是起點 吃了一次WA
這題應該有很多做法
我的構造法是 先把 $x$ 那排走完 再依序走 $1$ 到 $n$ 排
好好記錄走完上一排時在哪一格就可以了

C

給你長度為 $n$ 的 $a$ 序列跟長度為 $m$ 的 $b$ 序列

令 $c_i = a_i$ & $b_j$ (bitwise AND)

其中 $j$ 是由你決定的，且對於每個不同的 $i$ 你可以選重複的 $j$

請求出最小的 $c_{1} | c_{2} | c_{3} … | c_{n}$ (bitwise OR)

$(1 \leq n,m \leq 200, 0 \leq a_i, b_i < 2^9)$

這是基本的DP，狀態就 $dp(i, prefix) = min(dp(i + 1, prefix | (a_{i}$ & $b_{j})))$
prefix 就是前綴 bitwise OR 起來的狀態， i 代表說現在正在決定 $c_i$
不過賽中的時候忘了好好標記 vis 陣列 吃了一次 TLE

D

給你一個長度為 $n$ 的序列 $a$

你有 $n$ 個回合 每個回合你可以從序列中選擇一個元素並且將其從序列中刪除

被你選到的元素會對答案貢獻 $a_i$

如果被你選到的 $a_i > m$ 的話，你接下來會有 $d$ 個回合不能選元素

求最大化的答案是多少

$(1 \leq d \leq n \leq 10^5, 0 \leq m \leq 10^9, 0 \leq a_i \leq 10^9)$

敘述的有點爛 如果看不懂再點回去原題看吧

賽中一開始是中毒想用DP去做
後來才想到說先把序列分成兩組 一組 $> m$ ，一組 $\leq m$
然後枚舉我要取幾個 $> m$ 的元素，每次都拿其中最大的，用資結維護這件事
再用國小數學算一下在這樣的情況下
另外一組還能拿幾個元素，一樣盡量拿最大的，就可以收工了

後來發現這題會FST的原因是因為我把 $\leq$ 打成 $<$ 然後會過pretest也是有點神奇

E

給你一張有向帶邊權圖 $n$ 個點 $m$ 條邊

每條邊的邊權不會重複，且每個點的出度 $\leq k$

問你有多少個 tuple （長度 $k$，以 $c_1$ ~ $c_k$ 表示 ）滿足以下條件

對於每個點，先設其入度為 $d$ ，之後把那個點連接的所有邊刪掉除了邊權第 $c_d$ 小的那條邊

把所有該刪的邊刪完後，如果新的圖是一個長度為 $n$ 的有向環的話 那該 tuple 就符合條件

$(2 \leq n \leq 2 \cdot 10^5,2 \leq m \leq min(2 \cdot 10^5, n(n-1)),1 \leq k \leq 9, 1 \leq c_i \leq i)$

一樣先道歉我敘述的有點爛 如果看不懂再點回去原題看吧

這題會過pretest我後來也很問號，賽中時我把自己寫的解的複雜度算錯
我當下以為我的解是 $O(k^2n)$，送上去一開始TLE 壓常以後過了pretest
隔天發現我FST了 問了一下朋友才知道我是 $O(k!n)$ ，然後滿頭問號為什麼會過pretest
之後就被朋友爆雷正確的做法 這邊給上 $O(n + k! + m)$ 的做法

我原本的做法是枚舉 $k!$ 種 tuple 然後 $O(n)$ 檢查有向環
正確的做法是用Hash去檢查是不是有向環（Hash: 哈哈是我啦，你們是不是都以為有確定性的做法啊）
這樣就能透過預處理去算 我不太會解釋 我就附上官解連結吧
點我

後記

我寫這篇心得只是想分享我用各種奇怪方式過了 pretest
FST以後才發現自己錯爆，有點傻眼，有種後測的感覺
把 $\leq$ 打成 $<$ 是蠻大的失誤的
這場 E 也讓我理解到原來 Hash 還能這樣用啊
我現在還不夠強 大概率會開始強迫自己每天都要練題

想題是一件很快樂的事，但一直沒有好的成績時又有點焦慮
既然講好要打ICPC就要認真打 同時希望自己之後能更加享受比賽的樂趣。

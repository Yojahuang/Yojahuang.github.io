Codeforces Round 1398(Div2)

Author: Yojahuang Date: 八月 16, 2020  0:54:46 Category: 競程

A

給你一個長度為 $n$ 的序列 $a$

問你有沒有可能從該序列中挑出三個元素 使得那三個元素拼不出三角形

如果有 輸出那三個元素的編號

$(3 \leq n \leq 5 \cdot 10^4, 1 \leq a_i \leq 10^9, a_{i-1} \leq a_i)$

國小數學告訴我們 三角形兩邊長大於第三邊
這時候Greedy的想法就出現了
我們只要找最小的兩個元素當兩邊
最大的那個當第三邊，然後檢查有沒有大於第三邊，如果有代表那是三角形
sort一下 輕鬆沒煩惱
這題我在寫的時候 把 $\leq$ 打成 $<$ 了 好像跟上場犯的錯誤差不多

B

給你一個長度 $n$ 的 01字串

有兩個玩家正在進行一個遊戲 規則如下：

每回合 玩家會從字串中挑一段連續的 0 或 1 ，然後將其從字串中刪除

該玩家該回合的得分就是他挑到的那個subarray中 1的個數

問先手玩家採取最佳的情況下，最高能拿多少分

$(1 \leq n \leq 100)$

你會發現說 我們根本沒必要去拿連續的0
因為當你拿連續的0的時候，你可能會使得原本不相鄰的兩段1接在一起
這樣是做球給對方(?)
那既然我們確定要拿1了，顯然就是挑最長的連續1拿掉
這樣這題就做完了，你只要用迴圈算一下每段1的大小
丟進去vector之類的container，sort一下 每次取最大的就好

C

給你一個長度為 $n$ 的序列

定義 $\displaystyle sum(L,R) = \sum_{i=L}^{R}{a_i}$

請你輸出有多少 $sum(L,R)$ 符合

$sum(L,R) = R - L + 1$

$(1 \leq n \leq 10^5, 0 \leq a_i \leq 9)$

先弄出前綴和陣列 以下叫它 $S$
所以我們就等於說要算下面這個東西
$sum(l,r) = S_{r} - S_{l-1} = r - l + 1$
移項一下變成
$S_{r} - r = S_{l-1} - (l-1)$

這時候可以開一個map
對於每個 $i$ 把 $S_{i-1} - (i-1)$ 的個數紀錄起來
這樣這題就做完了

D

你有 $R$ 對 紅色棍子 $G$ 對綠色棍子 $B$ 對藍色棍子

每次你要挑 出兩對顏色不同的棍子 組成一個長方形

該長方形對於答案的貢獻是它的面積

問最大可能的答案

$(1 \leq R, G, B \leq 200)$

看完這題直覺會想說 那我一直挑最大的兩對出來不過好了
但請考慮以下測資

如果你選擇最大的兩對 答案只會是72
但如果你選 $7 \times 8 + 7 \times 9$ 的話，答案是119
所以這個直覺是錯的
既然沒有明確的選法，那就好好sort再DP就好囉

$dp(R, G, B) = max(dp(R-1,G-1,B) + r_{R} \times g_{G}, dp(R,G-1,B-1) + g_{G} \times b_{B}, dp(R-1,G,B-1) + r_{R} \times b_{B})$

F

https://codeforces.com/contest/1398/problem/F

我放棄敘述 直接貼連結

聽朋友說F其實不太難，今天去看了一下
在有tag提示的情況下想了半小時左右終於做出來了
但不確定我的複雜度是多少 有點難估

我是看到 tag 提示了二分搜以後才開竅的
對於每個 $i$ 當作起點 我用二分搜算出那個起點最遠可以拉到多遠 以下叫它 $w_i$

再來 我枚舉題目敘述的 $x$ 由小到大
維護一個 set 裡面是所有 $w_i \geq x$ 的 $i$
Greedy的想法說我們要讓起點越前面越好，所以就對set lower_bound 來更新下個起點
而該起點對於答案的貢獻就是 $\lfloor w_i / x \rfloor$ ，之後把起點更新到 $i + \lfloor w_i / x \rfloor \times x$

我自己估這樣做的複雜度是 $O(nlog^{2}n)$ 啦，不知道這樣估是不是好的
也不確定這樣到底是不是好的做法或者是我唬爛過這題的
